Compression via the PCA and the SVD¶

The main topic of this lesson is a data analysis method referred to as Principal Components Analysis (PCA). The method requires computing the eigenvectors of a certain matrix; one way to compute those eigenvectors is to use a special factorization from linear algebra called the Singular Value Decomposition (SVD).

This notebook is simply a collection of notes with a little bit of code to help illustrate the main ideas. It does not have any exercises that you need to submit. However, you should try to understand all the code steps that appear in the subsection entitled, Principal Components Analysis (PCA), as you will need to apply the SVD in a subsequent part of this assignment.

Motivation: data "compression." In previous lessons, we've looked at a few of the major tasks in data analysis: ranking, regression, classification, and clustering. Beyond these, the last problem you'll consider in our class is what we'll call compression.

At a high level, the term compression simply refers to finding any compact representation of the data. Such representations can help us in two ways. First, it can make the data set smaller and therefore faster to process or analyze. Secondly, choosing a clever representation can reveal hidden structure.

As a concrete example, consider the problem of dimensionality reduction: given a dd-dimensional data set, we wish to transform it into a smaller kk-dimensional data set where k≤dk≤d.

Choosing the kk dimensions in a clever way might even reveal structure that is hard to see in all dd original dimensions. For instance, look at the examples at the "visualizing PCA" website:

http://setosa.io/ev/principal-component-analysis/

Principal components analysis (PCA)¶

The method of principal components analysis (PCA) is one such technique. For this example, it would start by viewing these data as four (4) data points, one for each country, embedded in a 17-dimensional space (one dimension per food category). The following page will help build your intuition for PCA; the notes that then follow below show formally how PCA works and derives an algorithm to compute it.

http://setosa.io/ev/principal-component-analysis/

Basic definitions¶

Input data matrix, centered. Per our usual conventions, let x^0,…,x^m−1x^0,…,x^m−1 be th mm data points, where each xi∈Rdxi∈Rd is a single observation. Each observation is represented by a dd-dimensional real-valued vector corresponding to dd measured predictors. As usual, we can stack these into a data matrix, denoted X≡⎛⎝⎜⎜⎜x^T0⋮x^Tm−1⎞⎠⎟⎟⎟X≡(x^0T⋮x^m−1T).

However, we'll add one more important assumption: these data should be centered about their mean, i.e., 1m∑m−1i=0x^i=01m∑i=0m−1x^i=0. If the observations are not centered initially, then preprocess them accordingly.

Projections. Let φ∈Rdφ∈Rd be a vector of unit length, i.e., ∥φ∥22=φTφ=1‖φ‖22=φTφ=1. The projection of a data point x^ix^i onto φφ is x^Tiφx^iTφ, which measures the length of the projected vector.

The following code cell illustrates a projection. Given a vector x_hat and a line represented by a unit vector phi, it computes the projection x_hat_proj_phi of x_hat onto phi.

Maximizing projections¶

If the length of a projected data point is large, then intuitively, we have "preserved" its shape. So let's think of the total length of projections of all the data points as a measure of cost, which we can then try to maximmize.

Projection cost. Let J(φ)J(φ) be a cost function that is proportional to the mean squared projections of the data onto φφ:

J(ϕ)≡12m∑i=0m−1(x^Tiφ)2.J(ϕ)≡12m∑i=0m−1(x^iTφ)2.

The additional factor of "1/2" is for aesthetic reasons. (It cancels out later on.)

Let's also apply some algebra-fu to the right-hand side to put it into a more concise matrix form:

J(ϕ)==≡12φT(1m∑i=0m−1x^ix^Ti)φ12φT(1mXTX)φ12φTCφ.J(ϕ)=12φT(1m∑i=0m−1x^ix^iT)φ=12φT(1mXTX)φ≡12φTCφ.

In the last step, we defined C≡1mXTXC≡1mXTX. In statistics, if XX represents mean-centered data, then the matrix CC is also known as the sample covariance matrix of the data.

Principal components via maximizing projections. There are several ways to formulate the PCA problem. Here we consider the one based on maximizing projections.

Start by defining a principal component of the data XX to be a vector, φφ, of unit length that maximizes the sum of squared projections.

To convert this definition into a formal problem, there is a technique known as the method of Langrange multipliers, which may be applied to any minimization or maximization problem that has equality constraints. The idea is to modify the cost function in a certain way that effectively incorporates each constraint: for each constraint you will add to the cost function a term proportional to a dummy parameter times some form of the constraint.

Huh? It's easiest to see this formulation by example. In the case of a principal component, the modified cost function is

J^(φ,λ)≡J(φ)+λ2(1−φTφ),J^(φ,λ)≡J(φ)+λ2(1−φTφ),

where the second term captures the constraint: it introduces a dummy optimization parameter, λλ, times the constraint that φφ has unit length, i.e., ∥φ∥22=φTφ=1‖φ‖22=φTφ=1, or 1−φTφ=01−φTφ=0.

The reason to add the constraint in this way should become clear momentarily.

As before, the factor of "1/2" is there solely for aesthetic reasons and will "cancel out," as you'll soon see.

The optimization task is to find the φ∗φ∗ and λ∗λ∗ that maximize J^J^:

(φ∗,λ∗)≡argmaxφ,λJ^(φ,λ).(φ∗,λ∗)≡arg⁡maxφ,λJ^(φ,λ).

To solve this optimization problem, you just need to "take derivatives" of J^J^ with respect to φφ and λλ, and then set these derivatives to 0.

∇φJ^∂∂λJ^==Cφ−λφ12(1−φTφ).∇φJ^=Cφ−λφ∂∂λJ^=12(1−φTφ).

Setting these to zero and solving yields the following computational problem:

Cφ=1mXTXφ∥φ∥22==λφ1.Cφ=1mXTXφ=λφ‖φ‖22=1.

Is it now clear why the constraint was incorporated into J^J^ as it was? Doing so produces a second equation that exactly captures the constraint!

This problem is an eigenproblem, which is the task of computing an eigenvalue and its corresponding eigenvector of C=1mXTXC=1mXTX.

The matrix CC will usually have many eigenvalues and eigenvectors. So which one do you want? Plug the eigenvector back into the original cost function. Then, J(φ)=12φTCφ=λ2φTφ=λ2J(φ)=12φTCφ=λ2φTφ=λ2. In other words, to maximize J(φ)J(φ) you should pick the φφ with the largest eigenvalue λλ.

Finding an eigenpair via the SVD¶

So how do you find the eigenvectors of CC? That is, what algorithm will compute them?

One way is to form CC explicitly and then call an off-the-shelf eigensolver. However, forming CC explicitly from the data XX may be costly in time and storage, not to mention possibly less accurate. (Recall the condition number blow-up problem in the case of solving the normal equations.)

Instead, we can turn to the "Swiss Army knife" of linear algebra, which is the singular value decomposition, or SVD. It is an extremely versatile tool for simplifying linear algebra problems. It can also be somewhat expensive to compute accurately, but a lot of scientific and engineering effort has gone into building robust and reasonably efficient SVD algorithms. So let's assume these exist -- and they do in both Numpy and Scipy -- and use them accordingly.

The SVD. Every real-valued matrix X∈Rm×dX∈Rm×d has a singular value decomposition. Let s=min(m,d)s=min(m,d), i.e., the smaller of the number of rows or columns. Then the SVD of XX is the factorization, X=UΣVTX=UΣVT, where UU, ΣΣ, and VTVT are defined as follows.

The matrices U∈Rm×sU∈Rm×s and V∈Rd×sV∈Rd×s are orthogonal matrices, meaning UTU=IUTU=I and VTV=IVTV=I; and the matrix ΣΣ is an s×ss×s diagonal matrix.

Note that VV is taken to be d×sd×s, so that the VTVT that appears in UΣVTUΣVT is s×ds×d.)

The columns of UU are also known as the left singular vectors, and the columns of VV are the right singular vectors of XX. Using our usual "column-view" of a matrix, these vectors are denoted by uiui and vivi:

UV==[u0u1⋯us−1][v0v1⋯vs−1]U=[u0u1⋯us−1]V=[v0v1⋯vs−1]

Regarding the diagonal matrix ΣΣ, its entries are, collectively, called the singular values of XX:

⎡⎣⎢⎢⎢⎢⎢σ0σ1⋱σs−1⎤⎦⎥⎥⎥⎥⎥.[σ0σ1⋱σs−1].

From these definitions, the SVD implies that XV=UΣXV=UΣ. This form is just a compact way of writing down a system of independent vector equations,

Xvi=σiui.Xvi=σiui.

Recall that in PCA, you want to evaluate C=1mXTXC=1mXTX. In terms of the SVD,

XTX=VΣTUTUΣVT=VΣ2VT,XTX=VΣTUTUΣVT=VΣ2VT,

or

XTXV=VΣ2.XTXV=VΣ2.

This relation may in turn be rewritten as the system of vector equations,

XTXvi=σ2ivi.XTXvi=σi2vi.

In other words, every pair (φ,λ)≡(vi,σ2im)(φ,λ)≡(vi,σi2m) is a potential solution to the eigenproblem, Cφ=1mXTXφ=λφCφ=1mXTXφ=λφ. The pair with the largest eigenvalue is (v0,σ20m)(v0,σ02m).

Rank-kk approximations: the truncated SVD¶

We motivated PCA by asking for a single vector φφ, which effectively projects the data onto a one-dimensional subspace (i.e., a line). You might instead want to represent the original dd-dimensional data points on a kk-dimensional surface or subspace, where k≤s≤dk≤s≤d. As the previous discussion suggests, you could choose the top-kk right singular vectors of XX, v0,…,vk−1v0,…,vk−1.

Indeed, there is another "principled" reason for this choice.

Let A∈Rm×dA∈Rm×d be any matrix with an SVD given by A=UΣVTA=UΣVT. Per the notation above, let s≡min(m,d)s≡min(m,d).

Then, define the kk-truncated SVD as follows. Consider any k≤sk≤s, and let UkUk, ΣkΣk, and VkVk consist of the singular vectors and values corresponding to the kk largest singular values. That is, UkUk is the first kk columns of UU, VkVk is the first kk columns of VV, and ΣkΣk is the upper k×kk×k submatrix of ΣΣ. The kk-truncated SVD is the product UkΣkVTkUkΣkVkT.

Now consider the following alternative way to write the SVD:

A=UΣVT=∑i=0s−1uiσivTi.A=UΣVT=∑i=0s−1uiσiviT.

Each term, uiσivTiuiσiviT is known as a rank-11 product. So the existence of the SVD means that AA may be written as a sum of rank-1 products.

It would be natural to try to approximate AA by truncating the SVD after kk terms, i.e.,

A≈UkΣkVTk=∑i=0k−1uiσivTi.A≈UkΣkVkT=∑i=0k−1uiσiviT.

And in fact, there is no rank-kk approximation of AA that is better than this one!

In particular, consider any pair of kk column vectors, Yk∈Rm×kYk∈Rm×k and Zk∈Rd×kZk∈Rd×k; their product, YkZkYkZk has rank at most kk. Then there is a theorem that says the smallest difference between AA and the rank-kk product YkZkYkZk, measured in the Frobenius norm, is

minYk,Zk∥A−YkZTk∥2F=∥A−UkΣkVTk∥2F=σ2k+σ2k+1+σ2k+2+⋯+σ2s−1.minYk,Zk‖A−YkZkT‖F2=‖A−UkΣkVkT‖F2=σk2+σk+12+σk+22+⋯+σs−12.

In other words, the truncated SVD gives the best rank-kk approximation to AA in the Frobenius norm. Moreover, the error of the approximation is the sum of the squares of all the smallest s−ks−k singular values.

Applied to the covariance matrix, we may conclude that C=1mXTX≈1mVkΣ2kVTkC=1mXTX≈1mVkΣk2VkT is in fact the best rank-kk approximation of CC, which justifies choosing the kk eigenvectors corresponding to the top kk eigenvalues of CC as the principal components.

Summary: The PCA algorithm¶

Based on the preceding discussion, here is the basic algorithm to compute the PCA, given the data XX and the desired dimension kk of the subspace.

  1. If the data are not already centered, transform them so that they have a mean of 0 in all coordinates, i.e., 1m∑i=0m−1x^i=01m∑i=0m−1x^i=0.
  2. Compute the kk-truncated SVD, X≈UkΣkVTkX≈UkΣkVkT.
  3. Choose v0,v1,…,vk−1v0,v1,…,vk−1 to be the principal components.
